ConceptEngine

Turn any landing page into ready-to-use ad image prompts.

Before You Start

ConceptEngine is a skill for Claude Code — Anthropic's command-line tool for working with Claude. You'll need to install it first.

1 Install Homebrew (if you don't have it)

Open your terminal and run:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
2 Install Node.js
brew install node
3 Install Claude Code
npm install -g @anthropic-ai/claude-code
4 Sign in to Claude

Run claude in your terminal. It will open a browser window to sign in with your Anthropic account.

Once Claude Code is installed and you're signed in, you're ready to add ConceptEngine.

Installing ConceptEngine

1 Download the skill

Download concept-engine.zip (52MB — includes all reference images)

2 Unzip and move the folder

Unzip the file. Move the concept-engine folder to:

~/.claude/skills/

On Mac, the ~ means your home folder. If skills doesn't exist, create it.

3 Use it

Open Claude Code and type /concept-engine. That's it.

What This Does

You give it a landing page — either a URL or an HTML file on your computer. It reads the page, figures out who the product is for and what problems it solves, then writes image generation prompts based on 151 real ad examples.

Each prompt is designed to paste directly into an image generator like Midjourney, DALL-E, or Flux. You get back an HTML file with all the prompts, each with a copy button and a preview of the reference ad it's based on.

One-Time Setup

1 Put this folder in the right place

Copy this entire concept-engine folder into your Claude Code skills folder:

~/.claude/skills/

If the skills folder doesn't exist yet, create it. The ~ means your home folder — on Mac that's /Users/yourname/.

That's it for setup. The folder is about 52MB because it includes all 151 reference images — this means it works offline.

How to Use It

Starting the skill

Open your terminal and start Claude Code by typing claude. Once Claude is running, type:

/concept-engine

This tells Claude to use the ad concepts skill.

What Claude will ask you

After you run the command, Claude will ask for a landing page. You can give it:

  • A URL — like https://example.com/product
  • A file path — like /Users/you/Desktop/landing-page.html
You
/concept-engine
Claude
I'll generate ad concepts for you. What's the landing page URL or file path?
You
https://myproduct.com/sales-page

What happens next

Claude reads the landing page and does a few things:

  • Figures out the customer — who they are, what problem they have, what they want
  • Picks ad formats — selects 5-15 reference ads from the swipe file that fit this product
  • Writes prompts — creates image generation prompts based on each reference ad, using your product's copy and messaging
  • Creates an HTML file — puts everything in a single file you can open in your browser

This usually takes 2-5 minutes depending on the landing page length.

What you get

Your output file includes

📋

Copy-paste prompts

Each prompt has a button. Click it, the prompt is copied. Paste it into your image generator.

🖼️

Reference images

Above each prompt you'll see the real ad it's based on, so you know what style to expect.

📱

Mobile-friendly HTML

Dark theme, works on your phone if you want to generate images from there.

Claude will tell you where the file is saved. Open it in your browser to see all your prompts.

Tips

  • Better landing pages = better prompts. If the page clearly explains the product, problem, and customer, the prompts will be more specific.
  • You can ask for more. After the first batch, you can say "give me 5 more focused on social proof" or "more testimonial-style ads."
  • Product images help. If you have product photos in the same folder as your landing page, Claude can reference them in the prompts.

What's in This Folder

concept-engine/
  SKILL.md — the instructions Claude follows
  README.html — this file
  references/ — copywriting methodology and templates
  examples/ — example of a good prompt
  swipe-file/ — 151 reference ads + their schemas

You don't need to open or edit any of these files. They're there for Claude to read.

If Something Doesn't Work

Claude doesn't recognize the command: Make sure the folder is named exactly concept-engine and is inside ~/.claude/skills/. The full path should be:

/Users/yourname/.claude/skills/concept-engine/SKILL.md

Copy buttons don't work: If you're viewing the output file through a local server over HTTP (not HTTPS), the copy button uses a fallback method. It should still work, but if it doesn't, you can select the text manually and copy it.
